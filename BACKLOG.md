# Post-preview backlog

优先级按首版开发文档冻结：

1. 收集真实 Serial、Modbus、S7、MC 与 OPC UA 设备互操作结果，建立型号/固件测试矩阵。
2. 修复现场发现的协议兼容性、资源增长和长时间运行问题。
3. 扩展 S7/MC 设备系列、地址区域、符号访问及高级指令。
4. 增加 OPC UA browse、method、event、历史访问、冗余服务器和更完整的证书生命周期。
5. 为常见 IO、运动控制卡、扫码器和称重仪表创建独立厂商扩展包。
6. 增加 SQLite/数据库历史存储、索引查询、归档和迁移工具。
7. 增加 BenchmarkDotNet 基准、内存/句柄长期门禁、OpenTelemetry 指标和诊断命令。
8. 建立多操作系统与真实/虚拟串口 CI，验证 Linux、Windows 和目标硬件运行时。
9. 完成安全报告私密渠道、签名发布和软件物料清单。
10. API 稳定后发布 `1.0.0`，制定兼容性、弃用和长期支持策略。
