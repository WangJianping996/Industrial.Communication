# 故障排查

## 连接失败或超时

1. 记录 `CommunicationError.Code`、连接状态和安全的错误详情，不要只解析异常文本。
2. 确认 IP、端口、路由、防火墙、PLC 访问保护和并发连接上限。
3. 确认超时单位全部为 `TimeSpan`；现场首次联调可适当增大连接和请求超时。
4. TCP 远端关闭后通道会进入 `Faulted`。使用自动重连协调器，或显式断开后重新连接。
5. Serial 检查端口名、波特率、数据位、校验、停止位和握手配置是否完全一致。

## 请求超时但设备可能已执行写入

响应丢失不能证明设备没有执行写操作。非幂等写入、运动、回零、去皮和置零不应自动重试；先读取设备状态或使用协议事务机制确认。运动适配器从不在重连后自动重放动作。

## Modbus 地址偏移一位

本库使用零基协议地址。设备手册中的 `40001` 通常对应 `HR0`，但不同厂商手册表示法可能不同。先用单个已知寄存器验证，再配置批量映射。

## S7 无法访问 DB

首版仅支持绝对地址。S7-1200/1500 需要检查保护级别、PUT/GET 权限和数据块是否关闭优化访问。符号地址和优化 DB 不在首版范围。

## OPC UA 证书被拒绝

检查 rejected 目录中的服务器证书，核对主机名、有效期、用途和指纹后，将可信证书部署到 trusted 目录。不要把 `AllowUntrustedCertificates=true` 当作生产修复；启用它会产生 `SecurityWarning`。

## 收不到扫码或称重数据

确认帧类型与设备一致：分隔符、固定长度、长度字段或静默间隔。检查编码、终止符是否包含在帧中、最大帧长度及称重解析委托。扫码器的重复时间窗可能有意过滤连续相同条码。

## 队列积压或丢弃

确认 `QueueBackpressureStrategy`。`Wait` 会反压生产者；`Reject` 返回未接受；`DropNewest` 或 `DropOldest` 会明确报告丢弃。监控订阅默认使用有界队列保护通信链路，慢消费者应提高处理速度或调整容量。

## 资源没有释放

所有连接、客户端、服务器、订阅、适配器、监控器和队列都应通过 `await using` 或 `DisposeAsync` 释放。事件订阅由拥有者在释放时解除；自定义回调仍应避免长期捕获大型对象。
