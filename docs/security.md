# 安全说明

## 默认策略

- 消息监控默认不要求记录原始载荷；需要载荷时应配置允许字段和脱敏器。
- OPC UA 默认使用 `SignAndEncrypt` 与 `Basic256Sha256`，并拒绝未信任证书。
- 未信任 OPC UA 证书只能显式允许，且会触发安全警告；生产环境应维护 trust/rejected store。
- Modbus、S7comm 和 MC 3E 传统协议本身不提供现代身份认证或传输加密，应部署在隔离的工业网络、VPN 或安全网关之后。
- 自动重试只适合确认可重放的操作。运动、回零、使能、去皮、置零和其他危险动作必须显式调用。

## 机密和日志

- 密码、证书私钥、NuGet API Key 和真实设备凭据不得进入源码、示例、包、日志或测试数据。
- OPC UA 密码对象不应序列化或记录；生产程序应从安全 Secret Provider 注入。
- CI 的 `NUGET_API_KEY` 只从仓库 Secret 读取。工作流不会回显密钥。
- 发布审计拒绝 `.pfx`、`.p12`、`.pem`、`.key`、测试目录及私钥头。
- 日志中的换行和控制字符应编码或结构化记录，避免日志注入；面向用户的 `CommunicationError.Message` 不应包含凭据或完整敏感报文。

## 网络与文件

- Server 监听地址、最大连接数和请求队列容量必须显式评审；不要在不受控网络暴露工业端口。
- 历史和导出路径由可信配置提供。服务账户只授予目标目录的最小读写权限，并配置滚动与保留策略。
- 示例只使用 loopback、内存模拟器或占位地址，不包含真实设备地址。

发现安全问题时不要公开提交包含利用细节、密钥或现场拓扑的 Issue；应通过维护者公布的私密安全渠道报告。正式发布仓库建立后，需要在仓库 Security Policy 中补充准确联系人。
