状态说明：`已完成` 表示当前代码、测试和文档已经交付。

## 平台

| 项目 | 范围 | 状态 |
|---|---|---|
| NuGet 类库目标 | `netstandard2.1` 单目标 | 已完成 |
| 测试/示例宿主 | `net10.0`，不可打包 | 已完成 |
| 操作系统 | 任何具有兼容 .NET Standard 2.1 运行时的平台；串口名称和可用性受操作系统影响 | 已完成 |
| CPU 架构 | AnyCPU；厂商原生 SDK 不得进入核心包 | 已完成 |

## 当前已知限制

- Serial 实现已经完成，但 CI 没有虚拟串口对，发布前仍需在 Windows/Linux 串口或虚拟串口对上完成硬件集成验证。
- UDP 的 `ReceiveAsync` 在 .NET Standard 2.1 API 限制下通过关闭 Socket 来及时响应取消；取消接收后通道进入 `Faulted`，需要显式断开并重新连接。
- 分隔符解码器当前会复制待解析缓冲区，适合常规工业报文；大报文零拷贝优化列入后续性能工作。
- 不承诺覆盖所有 PLC 型号、固件、地址区或协议指令。
- S7-1200/1500 的访问能力受 PLC 侧保护级别、优化数据块等配置影响。
- MC 首版不包含 4E Frame、ASCII Frame、UDP 或全部软元件。
- OPC UA 使用独立协议包隔离官方 SDK；默认拒绝未信任证书。显式允许未信任证书时会产生安全警告。
- OPC UA 已通过内存会话验证读写、质量/时间戳映射和断线后的订阅重建；真实服务器、证书部署与互操作性仍需在目标现场验收。
- 通用设备适配器已覆盖数字 IO、运动控制、扫码器、称重设备，以及委托式私有 ASCII/二进制设备；运动、回零、使能等危险动作不会在重连后自动重放。
- 文件历史当前面向单进程实例，不提供多进程共享、数据库索引或静态加密。
- 慢监控订阅者的有界缓冲区满时会丢弃最旧观测，以保证主通信链路不被阻塞。
- 自动重试写操作可能在响应丢失时产生重复执行，非幂等写入应由调用方禁用重试或增加协议事务保护。
- Modbus 首版不包含功能码 22/23、ASCII 模式、RTU 广播客户端封装或 Modbus Security；传统 TCP/RTU 本身不提供认证与加密。
